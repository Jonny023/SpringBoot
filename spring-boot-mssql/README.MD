### 踩坑记录

#### 1. jdbc连接不上报错

> sqlserver 驱动程序无法通过使用安全套接字层(SSL)加密与 SQL Server 建立安全连接。错误:“sun.security.validator.ValidatorException: PKIX path building failed: sun.security.provider.certpath.SunCertPathBuilderException: unable to find valid certification path to requested target”。 ClientConnectionId:594c827e-0a80-42c0-9bc7-1b2eda7ff403

```yaml
#url: jdbc:sqlserver://localhost:1433;databaseName=test;trustServerCertificate=true
url: jdbc:sqlserver://localhost:1433;databaseName=test;encrypt=false
```

#### 2. 表名为关键字user处理

* 表对应实体添加`@TableName("[user]")`
* `mapper.xml`自定义sql查询时不能用`[user]`，只能用`SELECT * FROM "user"`
  * 原因好像是mybatis做sql转换时，如果是关键字会自动拼接`[]`，导致sql转换失败

#### 3. 低版本sqlserver分页时需要指定默认排序

> 本项目测试用的是sqlserver2022所以没有这样的问题

* 高版本用的是ROW_NUMBER()排序编号分页，无需指定分页条件，生成的分页sql为
```sql
SELECT
    TOP 2 *
FROM
    (
        SELECT
            ROW_NUMBER() OVER (ORDER BY RAND()) PAGE_ROW_NUMBER,
                *
        FROM
            (
                SELECT
                    *
                FROM "user"
            ) AS PAGE_TABLE_ALIAS
    ) AS PAGE_TABLE_ALIAS
WHERE
    PAGE_ROW_NUMBER > 0
ORDER BY
    PAGE_ROW_NUMBER
```

* 低版本，如sqlserver2000分页sql
```sql
SELECT *
FROM "user"
ORDER BY id
OFFSET 1 ROWS
FETCH NEXT 10 ROWS ONLY
```
